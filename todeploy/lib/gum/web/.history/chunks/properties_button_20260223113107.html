Mode

  • Single — one standalone button. The table is locked to one row.
  • Toolbar — several independent buttons; clicking one does not affect the others.
  • Radio group — mutually exclusive buttons; clicking one visually deselects all others.

Buttons table

  • Text — label shown inside the button.
  • Icon — optional Tabler icon name (e.g. home, power).
  • Color — Bulma color class applied to the button.
  • Enabled — whether the button is clickable at startup.

Size & Orientation

  • Size — base size; automatically scales on smaller screens.
  • Orientation — Horizontal: buttons side by side. Vertical: buttons stacked.

Actions for selected button

  • Change Device — sends a value to a device in an ExEn.
  • Execute Rule / Script — fires a rule or script by name.
  • Open URL — opens a URL in a new tab (value = the URL).

Multiple actions can be added; they all fire on the same click.

Behavior of selected button

  • Toggle — alternates between the action value (ON) and Value when OFF on successive clicks.
  • Confirmation — shows a dialog before firing. Leave Message empty for the default prompt.
  • Cooldown — disables the button for N seconds after clicking, preventing accidental double-firing.
  • Keyboard shortcut — triggers the button from the keyboard (e.g. Ctrl+1, Alt+S). Not fired when a text field has focus.
  • State reflection — highlights this button (using its color) when a watched device equals Active when value.
  • Conditional enable — automatically enables or disables this button based on a device value.
